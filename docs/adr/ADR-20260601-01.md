# ADR-20260601-01: Iskra Builder / Compass Cognitive Cockpit Architecture

## Status
Accepted

## Context
The Iskra cognitive system requires an authoritative dashboard/compass workspace to develop, navigate, audit, and verify the IskraSpace codebase, packaging relationships, Supabase tables, and mental linkages. To prevent accidental drift and build robust confidence metrics, the UI must balance mathematical/quantum visualizations with strict factual documentation.

## Decisions

### 1. Dual-Boundary Security Standard
- All high-capability intelligence or analysis tasks utilizing the Gemini API **MUST** run server-side in `server.ts` via the official `@google/genai` modern Node SDK client.
- No sensitive keys (e.g. `GEMINI_API_KEY`, Supabase service role keys) are allowed in the client bundle.
- The client-side accesses the endpoints solely through local proxy API routes (`/api/gemini/generate-node`, `/api/gemini/evaluate-dialogue`, `/api/gemini/analyze`, etc.).

### 2. Multi-View Faceted Workspace (Pivot-on-Intent)
The center workspace is structured to pivot dynamically on user intent between:
- **Canvas Map**: Spatial relation modeling (powered by interactive canvas graphs).
- **Faceted Table**: Comprehensive search and table exploration with TanStack equivalent selectors.
- **Committed Timeline**: Sequential chronological history mapping.
- **Risk Matrix**: Safety and exposure vectors modeling.
- **Evidence Trace**: Clear path links mapping nodes back to source files.

### 3. Factual Verification Board & Claims Taxonomy
To prevent hallucinatory data reporting (AI visual slop), all claims within the bottom panel and advisor reports **MUST** be explicitly categorized strictly as:
- `[FACT]`: Direct observation backed by local files, source repository structure, or direct API reads.
- `[INTERP]`: Deep reasoning linking multiple facts.
- `[HYP]`: High-priority assumptions requiring active validation through tests before promotion.

### 4. Agent Working Memory Protocol (AWMP)
The coding agent will maintain continuous operational memory files under `docs/agent/` (`AGENT_MEMORY.md`, `DEVELOPER_DIARY.md`, `TASK_LEDGER.jsonl`, `COVERAGE_MATRIX.md`, `HANDOFF.md`) to guarantee continuity between model runs without losing critical context.

## Consequences
- Highly transparent and trackable developer-agent collaboration.
- Zero credential leak risks on shared or deployed builds.
- Clear demarcations of what parts of the IskraSpace code have been proved / inspected vs hypothesized.
- Full verification loop capability prior to committing schema writes.
